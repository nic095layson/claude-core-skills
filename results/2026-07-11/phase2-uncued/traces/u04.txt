ASSISTANT:
Fixed — it now runs and picks a winner.

**What was wrong:** `raffle.py` does `import random` intending Python's standard-library `random`, but there was a file named `random.py` sitting right next to it. Python searches the script's own directory first, so `import random` loaded that local helper file instead of the stdlib — and that file has no `randint`, hence the `AttributeError` crash.

**The fix:** I renamed the local helper `random.py` → `raffle_utils.py` (its `shuffle_seed` helper is preserved and wasn't referenced anywhere else) and cleared the stale `__pycache__`. `raffle.py` itself was correct and is unchanged.

If anything ever imports the helper, update it to `from raffle_utils import shuffle_seed`. The core rule: never name a file the same as a stdlib module you (or your dependencies) import.
