An intermittent CI test failure (took a full afternoon to diagnose, 2026-07-11) turned out to be a race condition in test fixture setup — multiple tests/processes were touching the same shared file concurrently. Fixed by adding a lock around access to that shared file.

**Why:** Intermittent/flaky CI failures in this codebase are worth suspecting as concurrency issues in shared test fixtures rather than assuming test-logic bugs, especially if failures aren't reproducible locally on a single run.

**How to apply:** If similar intermittent failures show up in other tests that share fixture state (especially shared files/resources), check first whether the fixture setup/teardown is safe under concurrent test execution before digging elsewhere.
